# UI/UX Wireframes

Simple, clear interfaces that support the core game loop. Information when needed, immersion when possible.

---

## Design Principles

### Simplicity First
- Show only what's needed for the current decision
- Progressive disclosure - details on demand
- Consistent layout across phases

### The Core Loop is King
- Every screen should make the next action obvious
- No hunting for buttons
- Clear feedback on every action

### Respect the Player
- No dark patterns
- No hidden information that affects decisions
- Save anytime, quit anytime

## Mobile/Touch Considerations

If targeting tablets:
- Larger touch targets (44px minimum)
- Swipe to advance day
- Tap-and-hold for details
- Pinch to zoom on hex grid/map

## Simplicity Checklist

For every screen, verify:
- [ ] Can a new player understand what to do?
- [ ] Is the primary action obvious?
- [ ] Are consequences visible before committing?
- [ ] Can you return/undo if needed?
- [ ] Does it work without a tutorial?
